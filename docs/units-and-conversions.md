## Нормализация единиц

При вводе единица нормализуется к канонической форме:

1. Приводится к нижнему регистру и обрезаются пробелы
2. Проверяется вхождение в `CANONICAL_UNITS` — если есть, используется как есть
3. Проверяется в словаре `UNIT_ALIASES` — если есть, возвращается каноническое значение
4. Если ни то ни другое — выбрасывается ошибка валидации (HTTP 422)

Валидация происходит **дважды**:
- При входящем HTTP-запросе (`POST /v1/calculations`) — через Pydantic field_validator
- При обработке задачи в воркере — через `normalize_unit()`

## Таблица конверсий

| Из           | В            | Коэффициент | Пример                            |
|--------------|--------------|-------------|-----------------------------------|
| `kg`         | `t`          | 0.001       | 1000 кг × 0.001 = 1 т            |
| `t`          | `kg`         | 1000        | 1 т × 1000 = 1000 кг             |
| `m3`         | `l`          | 1000        | 1 м³ × 1000 = 1000 л             |
| `l`          | `m3`         | 0.001       | 1 л × 0.001 = 0.001 м³           |
| `m3`         | `m3_solid`   | 1           | 1 м³ природный = 1 м³ в массиве  |
| `m3_solid`   | `m3`         | 1           | 1 м³ в массиве = 1 м³ природный  |
| `m3`         | `m3_loose`   | 1.2         | 1 м³ в массиве × 1.2 = 1.2 м³ рыхлых |
| `m3_loose`   | `m3`         | 0.833       | 1.2 м³ рыхлых × 0.833 ≈ 1 м³    |
| `m3_solid`   | `m3_loose`   | 1.2         | Коэффициент разрыхления грунта   |
| `m3_loose`   | `m3_solid`   | 0.833       | Обратный коэффициент             |
| `bag`        | `kg`         | 25          | 1 мешок цемента = 25 кг          |
| `kg`         | `bag`        | 0.04        | 1 кг = 0.04 мешка                |

**Примечание по земляным работам:** Коэффициент разрыхления грунта принят равным **1.2** (среднее значение для суглинков и супесей). Для отдельных типов грунта этот коэффициент может отличаться.

**Примечание по мешкам:** Принят стандартный вес мешка цемента = 25 кг. Для других материалов вес мешка может отличаться.

## Правило unpriced

Если для позиции спецификации **невозможно найти цену ни через одну конверсию**, позиция помечается как `unpriced`:

- `pricing_method: "unpriced"`
- `confidence: "none"`
- `unit_price: 0`
- `line_total: 0`
- `fallback_reason: "no_price_found"`

Конверсия считается **невозможной** если:
- Пара `(from_unit, to_unit)` отсутствует в таблице `UNIT_CONVERSIONS`
- Цены не найдены ни в одном конвертируемом варианте единицы

Например, перевести `trip` в `m3` нельзя — это физически разные размерности. Позиции с такими единицами могут получить цену только через exact match или category fallback.
